# 06 · 外网访问(ngrok 教程)

> 默认 Docker 镜像 `edik/vnc-research` 只监听 6080,只能内网访问。
> 如需从公网访问 → 用 ngrok 把 6080 暴露到 https 隧道。

**前提**:你已读过 [docs/05-安全考量.md](05-安全考量.md),明白"不鉴权 = 灾难"。

---

## 何时需要

- 你的 vnc-research 容器跑在**没有公网 IP 的服务器**上(内网 / 家里 / 公司防火墙后)
- 你想**出差/通勤时**从手机/笔记本访问 VNC 浏览器
- 你想**临时分享**给同事看屏幕(用完就关)

不需要公网访问 → 跳过本文,内网用 6080 就够了。

---

## 安装 ngrok

**Debian / Ubuntu**:

```bash
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
    | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null 2>&1
echo 'deb https://ngrok-agent.s3.amazonaws.com buster main' \
    | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok
```

**macOS**:

```bash
brew install ngrok/ngrok/ngrok
```

**Windows**:

```powershell
choco install ngrok
# 或去 https://ngrok.com/download 下载安装
```

**验证**:

```bash
ngrok version
# ngrok version 3.x.x
```

---

## 注册账号 + authtoken(免费)

- 访问 https://dashboard.ngrok.com/signup 注册
- 在 dashboard 拿到 `authtoken`(长得像 `2abc...xyz`)
- 配置到本机:

```bash
ngrok config add-authtoken <你的-token>
```

写到 `~/.config/ngrok/ngrok.yml`(macOS 是 `~/Library/Application Support/ngrok/ngrok.yml`)。

> **不注册也能用**,只是每次启动域名随机 + 限速更严。注册免费,强烈建议注册。

---

## 启动 ngrok 隧道

**前置**:vnc-research 容器已经在跑(见 [docs/02-部署指南.md](02-部署指南.md))。

```bash
ngrok http 6080
```

ngrok 会输出形如:

```
Forwarding  https://xxxx-xxxx.ngrok-free.app -> http://localhost:6080
```

**复制那个 `https://xxxx-xxxx.ngrok-free.app` URL** → 浏览器打开就是 noVNC 页面。

> 按 `Ctrl+C` 关闭隧道。建议**用完就关**,不要长期挂公网。

---

## 鉴权(强烈建议)

ngrok 默认**不鉴权** → 任何扫到 URL 的人都能操控你的浏览器。

### 单层鉴权:ngrok basic-auth

**ngrok basic-auth** — 弹窗要用户名密码:

```bash
ngrok http 6080 --basic-auth 用户名:密码
```

(把 `用户名` / `密码` 换成你想要的)

单层鉴权就够,ngrok basic-auth 防自动化扫描;**两层不是必须**(用户必须先知道 basic-auth)。

⚠️ **不鉴权 = 灾难**:任何扫到 ngrok 链接的人都能操控你的浏览器 → 用你的 cookie 登录的账号写评论 / 发消息 / 改密码。

---

## 区域选择(国内用户)

ngrok 默认连 `us` 节点,国内访问慢。设区域 env:

```bash
# 选亚太节点
ngrok http 6080 --region ap
```

可用区域:`us`(美国) / `eu`(欧洲) / `ap`(亚太,新加坡) / `au`(澳洲) / `sa`(南美) / `jp`(日本) / `in`(印度)。

国内推荐 `ap` 或 `jp`。

---

## 持久化配置(可选)

每次输 `--basic-auth` 麻烦,可以写到 `~/.config/ngrok/ngrok.yml`:

```yaml
version: "2"
authtoken: <你的-token>
tunnels:
  vnc:
    proto: http
    addr: 6080
    basic_auth:
      - 用户名:密码
    region: ap
```

然后一行启动:

```bash
ngrok start vnc
```

## 关闭隧道

`Ctrl+C` 即可。建议用完就关,不要长期挂公网。

## 故障排查

### `ngrok: command not found`

没装,见 [安装 ngrok](#安装-ngrok) 章节。

### `tunnel session failed`

- authtoken 没配 / 失效 → `ngrok config add-authtoken <token>`
- free plan 限速触发 → 等几秒重试
- 网络问题(国内连 ngrok 偶尔抽风)→ 换 `--region ap` 或 `jp` 再试

### 访问 ngrok URL 显示 "Tunnel Not Found"

ngrok 进程已退出 → 重新跑 `ngrok http 6080`。

### 不弹 basic-auth 窗

- 确认启动命令带了 `--basic-auth 用户名:密码`
- 检查用户名密码里有特殊字符 → 试简单点的(纯字母数字)

### 国内连 ngrok 太慢

设 `--region ap`(亚太,新加坡)或 `jp`(日本)。

### 进入 noVNC 后看不到桌面

- 浏览器 F5 刷新几次(noVNC 首次连接偶发渲染失败)
- 看 [07-故障排查](07-故障排查.md) · 7.1 容器日志 / 7.2 端口检查
