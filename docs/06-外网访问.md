# 06 · 外网访问(ngrok + 鉴权)

> ⚠️ **警告**:外网访问会把你的 VNC 浏览器暴露到公网,**必须配鉴权**才能用。
> 不鉴权 = 任何扫到 ngrok 链接的人都能操控你的浏览器 = 账号被封 / 隐私泄漏。
> 阅读本文前请确认你已读过 [docs/05-安全考量.md](05-安全考量.md)。
> 默认推荐方式(纯内网)见 [docs/00-快速开始.md](00-快速开始.md);本文讲的是**例外场景**——你必须从公网访问。

---

## 1. 风险提示(必读)

### 1.1 没有鉴权会怎样

- 任何人扫到 ngrok 链接 → 直接打开 noVNC → 看你的浏览器 → 用你的 cookie 登录的小红书 / 抖音 / 知乎等账号 → 写评论 / 发消息 / 改密码 / 转账
- ngrok 链接一旦泄漏(微信群、GitHub commit、截图),**无法"撤回"**
- VNC 浏览器里的所有 cookie / localStorage / 已登录账号 = **直接被人接管**

### 1.2 必须做的三件事

- [ ] **VNC 密码**([`scripts/start-all.sh`](../scripts/start-all.sh) 默认生成 8 位数字密码,启动日志打印;用户可用 `export VNC_PASSWORD=你想要的值` 然后启动 start-all.sh 覆盖)
- [ ] **ngrok basic-auth**(自己设 `export NGROK_USER=用户名` + `export NGROK_PASS=密码`)
- [ ] **Docker 配置**(如果你用 Docker,镜像里**不能 hardcode 密码**,必须 env 注入)

### 1.3 推荐但不强制

- ngrok 注册账号 + authtoken(免费,可保留配置 + 更稳定)
- ngrok Access / OAuth(企业级)
- 限制 VNC 只能查看不能操作(只读模式)

### 1.4 ngrok 默认行为(本次决策)

| 项 | 默认值 | 说明 |
|---|---|---|
| 鉴权 | **全可选** | 文档说明,**不强制**(由用户自己设) |
| VNC 密码 | **自动生成 8 位数字** | 启动日志打印,用户可用 `VNC_PASSWORD` env 覆盖 |
| ngrok basic-auth 用户名密码 | **默认无** | 用户可用 `NGROK_USER` + `NGROK_PASS` 自设 |
| ngrok authtoken | **默认空(free anonymous)** | 用户注册后可填 `NGROK_AUTHTOKEN` |
| ngrok 隧道类型 | **默认 HTTP** | 也提供 TCP 备用 |

## 2. 准备工作

### 2.1 安装 ngrok

**Linux(Debian / Ubuntu)**:

```bash
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
    | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null 2>&1
echo 'deb https://ngrok-agent.s3.amazonaws.com buster main' \
    | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok
```

**macOS**:

```bash
brew install ngrok/ngrok/ngrok
```

**Windows**:

```powershell
choco install ngrok
# 或去 https://ngrok.com/download 装
```

**验证安装**:

```bash
ngrok version
# ngrok version 3.x.x 之类
```

### 2.2 注册 ngrok 账号(可选,但推荐)

- 访问 https://dashboard.ngrok.com/signup 注册(免费)
- 在 dashboard 拿到 `authtoken`(长得像 `2abc...xyz`)
- 配置到本机:

  ```bash
  ngrok config add-authtoken <你的-token>
  ```

  写到 `~/.config/ngrok/ngrok.yml`(或 `~/Library/Application Support/ngrok/ngrok.yml`)
- **不注册也能用**,只是每次启动域名随机 + 限速更严;注册后 authtoken 稳定保留

## 3. 方案 A(主推)· ngrok HTTP 隧道 + 双层鉴权

> **双层鉴权** = ngrok basic-auth(防扫描) + VNC 密码(防 basic-auth 被突破)。

### 3.1 架构

```
┌─────────────────┐   HTTPS   ┌──────────────────┐
│  外网用户浏览器  │ ────────→ │  ngrok 公网入口   │
│ (anywhere)      │ ←───────  │  (basic-auth 弹窗)│
└─────────────────┘           └──────────────────┘
                                       │ 隧道(WebSocket)
                                       ▼
                              ┌──────────────────┐
                              │ 你的服务器       │
                              │ noVNC :6080      │
                              │   ↓              │
                              │ x11vnc :5900     │  ←── VNC 密码弹窗
                              │   ↓              │
                              │ chromium         │
                              └──────────────────┘
```

### 3.2 启动步骤

1. 启动 vnc-research 整套(已含 VNC 密码):

   ```bash
   cd /root/project/docs/vnc-research
   ./scripts/start-all.sh
   ```

   看启动日志里的 `🔐 VNC 密码(本次启动自动生成):12345678`,**记下来**(就是 8 位数字)

2. 启动 ngrok 隧道(带 basic-auth):

   ```bash
   export NGROK_USER="你的用户名"
   export NGROK_PASS="$(openssl rand -base64 16)"
   echo "你的 ngrok 密码: $NGROK_PASS"
   ./scripts/start-ngrok.sh
   ```

3. 看 ngrok 输出,拿到形如 `https://xxxx.ngrok-free.app` 的 URL

4. 外网浏览器打开 URL:
   - **第一步**:ngrok 弹 basic-auth 窗 → 输入 `NGROK_USER` / `NGROK_PASS`
   - **第二步**:进入 noVNC 页面 → 点 "Connect"
   - **第三步**:noVNC 弹 VNC 密码窗 → 输入日志里那 8 位数字
   - **第四步**:进入 chromium,开始操作

### 3.3 配置项(全部可选,env 注入)

| env | 默认值 | 含义 |
|---|---|---|
| `NGROK_AUTHTOKEN` | 空 | ngrok authtoken,设了能保留配置 + 更稳定 |
| `NGROK_USER` | 空 | ngrok basic-auth 用户名,**强烈建议设** |
| `NGROK_PASS` | 空 | ngrok basic-auth 密码,**强烈建议设** |
| `NGROK_REGION` | `us` | ngrok 区域(us / eu / ap / au / sa / jp),国内用户用 `ap` 或 `jp` 更快 |
| `NGROK_DOMAIN` | 空 | 自定义域名(**付费 plan** 才能用) |
| `NGROK_TCP` | `0` | 设 `1` 切到 TCP 隧道(给原生 VNC 客户端用) |
| `VNC_PASSWORD` | 空(自动生成 8 位数字) | VNC 密码,设了就用你设的 |

### 3.4 示例:开启 basic-auth + 自定义区域

```bash
# 1. 设置 basic-auth
export NGROK_USER="edik"
export NGROK_PASS="$(openssl rand -base64 16)"
echo "你的 ngrok 密码(复制保存): $NGROK_PASS"

# 2. 选近的区域(国内用户)
export NGROK_REGION="ap"

# 3. 启动
./scripts/start-ngrok.sh
```

输出类似:

```
[1/3] ...
[2/3] 启动 ngrok 隧道 ...
    模式:HTTP(浏览器 noVNC,端口 6080)
    basic-auth:已启用(用户名=edik)
[3/3] 启动中 ...

📌 提示:启动后会输出 ngrok URL,复制那个 URL 给外网用户即可

Session Status   online
Account          your-email@example.com (Plan: Free)
Version          3.x.x
Region           ap (Asia/Pacific)
Latency          45ms
Web Interface    http://127.0.0.1:4040
Forwarding       https://xxxx-xxxx-xxxx.ap.ngrok-free.app -> http://localhost:6080

Connections      ttl     opn     rt1     rt5     p50     p90
                0       0       0.00    0.00    0.00    0.00
```

复制 `Forwarding` 那行的 `https://...` 给外网用户。

## 4. 方案 B(备选)· ngrok TCP 隧道

> 适合**原生 VNC 客户端**(不是 noVNC 浏览器),端口 5900 暴露。
> 性能比 HTTP 方案稍好(不走浏览器渲染),但**配置麻烦 + 缺 basic-auth 概念**。

### 4.1 用法

```bash
# 注册 ngrok 账号 + authtoken(TCP 隧道免费 plan 也能用)
# 把 <your-token> 换成你 dashboard 里拿到的实际 token
export NGROK_AUTHTOKEN="<your-token>"
# 切到 TCP 模式
export NGROK_TCP=1

# 启动
./scripts/start-ngrok.sh
```

ngrok 会输出形如:

```
Forwarding   tcp://0.tcp.ngrok.io:12345 -> localhost:5900
```

用 VNC viewer(RealVNC / TigerVNC / Remmina 等)连 `0.tcp.ngrok.io:12345`,输 VNC 密码(就是 `start-all.sh` 启动日志里那 8 位数字)。

### 4.2 优缺点

- ✅ 原生 VNC 性能更好(不经浏览器渲染)
- ❌ **没有 basic-auth 概念**(TCP 隧道),只能靠 VNC 密码 + ngrok 防火墙规则
- ❌ 需要装 VNC viewer 客户端(RealVNC / TigerVNC)
- ❌ 部分网络环境对 TCP 隧道不友好

### 4.3 适合场景

- 你有现成的 VNC 客户端习惯(不想用浏览器)
- 追求低延迟(画屏刷得快)
- **不推荐给普通用户用**

## 5. Docker 部署 + ngrok 配置

> 在 Docker 镜像里跑 ngrok 时,**密码不能写死**(写死 = 镜像泄漏 = 密码泄漏)。
> 必须 env 注入。

### 5.1 docker run 示例

```bash
# 准备密码(不要 commit 到 git)
# 生成 8 位数字 VNC 密码
export VNC_PASSWORD="$(shuf -i 10000000-99999999 -n 1)"
export NGROK_USER="edik"
export NGROK_PASS="$(openssl rand -base64 16)"

echo "VNC 密码: $VNC_PASSWORD"
echo "ngrok 密码: $NGROK_PASS"

# 启动(假设镜像已 build 好,叫 vnc-research)
docker run -d \
    --name vnc-research \
    --restart unless-stopped \
    -p 6080:6080 \
    -p 9222:9222 \
    -e VNC_PASSWORD="$VNC_PASSWORD" \
    -e NGROK_AUTHTOKEN="$NGROK_AUTHTOKEN" \
    -e NGROK_USER="$NGROK_USER" \
    -e NGROK_PASS="$NGROK_PASS" \
    -e NGROK_REGION="${NGROK_REGION:-ap}" \
    vnc-research
```

⚠️ **不要**这样写(`docker run -e` 写密码):

```bash
# 错:密码写在命令行 → history 留下 / docker inspect 能看
docker run -e VNC_PASSWORD=12345678 ...
```

✅ **推荐**:用 `--env-file` 或 docker secret / `.env` 文件。

### 5.2 docker-compose.yml 示例

```yaml
# docker-compose.yml
version: "3.8"

services:
  vnc-research:
    build: .
    image: vnc-research:latest
    container_name: vnc-research
    restart: unless-stopped
    ports:
      - "6080:6080"
      - "9222:9222"
    env_file:
      - .env   # ← VNC_PASSWORD / NGROK_USER / NGROK_PASS 都在 .env 里
    volumes:
      - chrome-data:/home/vncuser/.config/chromium

volumes:
  chrome-data:
```

配套 `.env`(加入 `.gitignore`!):

```bash
# .env(别 commit!)
VNC_PASSWORD=12345678
NGROK_USER=edik
NGROK_PASS=随机的base64字符串
NGROK_AUTHTOKEN=你的authtoken
NGROK_REGION=ap
```

启动:

```bash
docker compose up -d
# 进容器手动起 ngrok(可选)
docker exec -it vnc-research /usr/local/bin/start-ngrok.sh
```

### 5.3 Dockerfile 提示(如果还没有)

```dockerfile
FROM debian:13-slim

# 装系统包
RUN apt-get update && apt-get install -y \
        x11vnc novnc websockify x11-apps chromium \
        python3 python3-pip curl ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# 装 ngrok(Debian/Ubuntu 源)
RUN curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
        | tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null 2>&1 \
    && echo 'deb https://ngrok-agent.s3.amazonaws.com buster main' \
        | tee /etc/apt/sources.list.d/ngrok.list \
    && apt-get update && apt-get install -y ngrok \
    && rm -rf /var/lib/apt/lists/*

# Python 依赖
RUN pip3 install --break-system-packages playwright

# 用户
RUN useradd -m -s /bin/bash vncuser

# 脚本
COPY scripts/ /usr/local/bin/
RUN chmod +x /usr/local/bin/*.sh

USER vncuser
WORKDIR /home/vncuser

EXPOSE 6080 9222

# 默认只跑 vnc-research;ngrok 按需手动 docker exec 起
CMD ["/usr/local/bin/start-all.sh"]
```

## 6. 安全 checklist(发布前必过)

发布 ngrok URL 之前,**逐条核对**:

- [ ] **VNC 密码已启用**(看 [`start-all.sh`](../scripts/start-all.sh) 启动日志,确认 `🔐 VNC 密码` 这一行打印了 8 位数字)
- [ ] **ngrok basic-auth 已设**(看 [`start-ngrok.sh`](../scripts/start-ngrok.sh) 启动日志,确认 `basic-auth:已启用` 这一行)
- [ ] **没用 default password**(密码不是 `password` / `12345678` / `vnc` 这类)
- [ ] **没把 ngrok URL 发到公开群**(微信群也要小心,可能被截图外传)
- [ ] **用完就关 ngrok 隧道**(`Ctrl+C` 即可,不留窗口过夜)
- [ ] **浏览器里的小红书 / 抖音账号登出过 VNC 浏览器**(用普通浏览器,避免 cookie 共享)
- [ ] **authtoken 来自你自己的 ngrok 账号**(不是借用别人的)
- [ ] **Docker 部署用 `--env-file` / `.env`,不用命令行写密码**
- [ ] **`.env` 在 `.gitignore` 里**(密码不跟代码一起 commit)

## 7. 故障排查

### 7.1 ngrok 输出 "command not found"

- 检查 `which ngrok`,没装就装(见 [2.1](#21-安装-ngrok))

### 7.2 ngrok 输出 "tunnel session failed"

- authtoken 没配 / 失效 → `ngrok config add-authtoken <token>`
- free plan 限速触发 → 等几秒重试
- 网络问题(国内连 ngrok 偶尔抽风)→ 换 `NGROK_REGION=ap` 或 `jp` 再试

### 7.3 访问 ngrok URL 显示 "Tunnel Not Found"

- ngrok 进程已退出 → 重启 [`./scripts/start-ngrok.sh`](../scripts/start-ngrok.sh)

### 7.4 访问 ngrok URL 不弹 basic-auth

- 检查 `NGROK_USER` / `NGROK_PASS` env 是否在 [`start-ngrok.sh`](../scripts/start-ngrok.sh) 同 shell 里 export
- 看 ngrok 启动日志,确认 `basic-auth:已启用(用户名=...)` 这一行

### 7.5 进入 noVNC 后输 VNC 密码报错

- 看 [`start-all.sh`](../scripts/start-all.sh) 启动日志,`🔐 VNC 密码` 那一行就是 8 位数字
- 如果密码丢了,**重启** `start-all.sh` 会重新生成

### 7.6 free plan 每次启动域名都变

- 注册 ngrok 账号 + authtoken(免费)→ 域名虽然还是随机,但 authtoken 稳定保留配置
- 真要固定域名要付费 plan

### 7.7 启动 ngrok 后浏览器报 ERR_NGROK_3200 / Tunnel Not Found

- ngrok 没连上 → 看 ngrok 输出,可能账号被封 / 限速 / 区域不对
- 换 `NGROK_REGION` 再试

### 7.8 国内连 ngrok 太慢

- 设 `export NGROK_REGION=ap`(亚太,新加坡节点)
- 或用 `jp`(日本节点)
- 实在不行考虑 cloudflared 替代(超出本项目范围)
