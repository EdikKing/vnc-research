# 03 · 外网访问(ngrok 教程)

> 默认 Docker 镜像 `edik/vnc-research` 只监听 6080,只能内网访问。
> 如需从公网访问 → 用 ngrok 把 6080 暴露到 https 隧道。

> ⚠️ **警告**:不鉴权 = 灾难(见下文"鉴权与安全"章节)。

---

## 何时需要

- 你的 vnc-research 容器跑在**没有公网 IP 的服务器**上(内网 / 家里 / 公司防火墙后)
- 你想**出差/通勤时**从手机/笔记本访问 VNC 浏览器
- 你想**临时分享**给同事看屏幕(用完就关)

不需要公网访问 → 跳过本文,内网用 6080 就够了。

---

## 安装 ngrok

**Debian / Ubuntu**:

```bash
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
    | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null 2>&1
echo 'deb https://ngrok-agent.s3.amazonaws.com bullseye main' \
    | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok
```

**macOS**:

```bash
brew install ngrok/ngrok/ngrok
```

**Windows**:

```powershell
choco install ngrok
# 或去 https://ngrok.com/download 下载安装
```

**验证**:

```bash
ngrok version
# ngrok version 3.x.x
```

## 注册账号 + authtoken(免费)

- 访问 https://dashboard.ngrok.com/signup 注册
- 在 dashboard 拿到 `authtoken`(长得像 `2abc...xyz`)
- 配置到本机:

```bash
ngrok config add-authtoken <你的-token>
```

写到 `~/.config/ngrok/ngrok.yml`(macOS 是 `~/Library/Application Support/ngrok/ngrok.yml`)。

> **不注册也能用**,只是每次启动域名随机 + 限速更严。注册免费,强烈建议注册。

---

## 启动 ngrok 隧道

**前置**:vnc-research 容器已经在跑(见 [docs/01-部署指南.md](01-部署指南.md))。

```bash
ngrok http 6080
```

ngrok 会输出形如:

```
Forwarding  https://xxxx-xxxx.ngrok-free.app -> http://localhost:6080
```

**复制那个 `https://xxxx-xxxx.ngrok-free.app` URL** → 浏览器打开就是 noVNC 页面。

> 按 `Ctrl+C` 关闭隧道。建议**用完就关**,不要长期挂公网。

---

## 鉴权与安全

⚠️ **公网访问 = 暴露账号风险**:任何扫到 ngrok 链接的人,都能操控你浏览器里的所有内容 → 用你的 cookie 写评论 / 发消息 / 改密码。

**唯一防护**:ngrok basic-auth(浏览器弹窗要用户名密码)。**必须启用**。

### 启用 basic-auth(必须)

```bash
ngrok http 6080 --basic-auth 用户名:密码
```

(把 `用户名` / `密码` 换成你想要的)

不传 `--basic-auth` = 任何人扫到 URL 都能直接打开 noVNC 操控你的浏览器,**等于没设防**。

### 强密码建议

- ✅ **至少 12 位**,包含大小写字母 + 数字 + 特殊字符
- ✅ 用密码管理器生成(1Password / Bitwarden / KeePass)
- ❌ **不要**用 `admin / 12345678 / password` 这种
- ❌ **不要**用跟其他账号相同的密码(ngrok URL 一旦泄漏,密码也泄漏)

### 链接泄漏风险(ngrok URL 安全)

ngrok free plan 的 URL 形如 `https://xxxx-xxxx.ngrok-free.app` —— **这个 URL 本身就是凭证**。

**避免泄漏渠道**:
- ❌ **不要**把 ngrok URL 发到公开群(微信群 / Slack / 邮件抄送)
- ❌ **不要**截屏发给别人(URL 永久可见)
- ❌ **不要**commit 到 git / 写到文档 / 贴到 issue
- ✅ 用密码管理器 + 加密分享工具(Bitwarden Send 等)
- ✅ **用完就关**:`Ctrl+C` 关闭隧道,URL 立即失效(free plan URL 永久但关了就拒绝连接)

### 多人协作场景

如果多人要用,不要共享一个 basic-auth:
- 给每个人单独创建 ngrok 账号 + 单独的 basic-auth
- 不要在群里贴 ngrok URL + 密码

### 关闭后清理

`Ctrl+C` 关闭 ngrok 后:
- ✅ basic-auth 凭证失效
- ✅ URL 失效(free plan 也立即拒绝连接)
- ⚠️ VNC 容器还在跑(还在监听 6080),但**没有 ngrok 转发**,公网访问不到

## 区域选择(国内用户)

ngrok 默认连 `us` 节点,国内访问慢。设区域 env:

```bash
# 选亚太节点
ngrok http 6080 --region ap
```

可用区域:`us`(美国) / `eu`(欧洲) / `ap`(亚太,新加坡) / `au`(澳洲) / `sa`(南美) / `jp`(日本) / `in`(印度)。

国内推荐 `ap` 或 `jp`。

---

## 持久化配置(可选)

每次输 `--basic-auth` 麻烦,可以写到 `~/.config/ngrok/ngrok.yml`:

```yaml
version: "2"
authtoken: <你的-token>
tunnels:
  vnc:
    proto: http
    addr: 6080
    basic_auth:
      - 用户名:密码
    region: ap
```

然后一行启动:

```bash
ngrok start vnc
```

## 故障排查

### `ngrok: command not found`

没装,见 [安装 ngrok](#安装-ngrok) 章节。

### `tunnel session failed`

- authtoken 没配 / 失效 → `ngrok config add-authtoken <token>`
- free plan 限速触发 → 等几秒重试
- 网络问题(国内连 ngrok 偶尔抽风)→ 换 `--region ap` 或 `jp` 再试

### 访问 ngrok URL 显示 "Tunnel Not Found"

ngrok 进程已退出 → 重新跑 `ngrok http 6080`。

### 不弹 basic-auth 窗

- 确认启动命令带了 `--basic-auth 用户名:密码`
- 检查用户名密码里有特殊字符 → 试简单点的(纯字母数字)

### 国内连 ngrok 太慢

设 `--region ap`(亚太,新加坡)或 `jp`(日本)。

### 进入 noVNC 后看不到桌面

- 浏览器 F5 刷新几次(noVNC 首次连接偶发渲染失败)
- 看 `docker logs vnc-research` 查容器日志
