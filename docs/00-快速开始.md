# 快速开始 · 5 分钟跑起来

> vnc-research 通过 Docker Hub 镜像 [edik/vnc-research](https://hub.docker.com/r/edik/vnc-research) 一键启动。
> 镜像里已经装好 Xvfb / x11vnc / noVNC / chromium / playwright,无需任何宿主机依赖(只要 Docker)。

---

## Step 0:环境要求(1 分钟确认)

- Docker 引擎(版本 ≥ 20.10):`docker --version`
- 浏览器(任意)
- 至少 1GB 可用内存
- Linux 宿主机推荐(macOS / Windows 需要 Docker Desktop)

---

## Step 1:启动容器(10 秒)

```bash
docker run -d --name vnc-research -p 6080:6080 -p 9222:9222 edik/vnc-research
```

输出形如:
```bash
abc123def456...   (容器 ID)
```

✅ 容器已经在后台跑。

---

## Step 2:等启动 + 看日志(20 秒)

```bash
docker logs -f vnc-research
```

找到这一行就是 8 位 VNC 密码:
```bash
🔐 VNC 密码(本次启动自动生成):12345678
```

**记下来**(等下浏览器弹窗要输)。`Ctrl+C` 退出日志(容器还在跑)。

---

## Step 3:浏览器访问 VNC(2 秒)

打开:http://localhost:6080/vnc.html

如果服务器不在本机,把 localhost 换成服务器 IP。

noVNC 会弹 VNC 密码窗 → 输刚才的 8 位数字 → 看到 chromium 桌面 = 成功。

---

## Step 4:让 AI Agent 连进来(30 秒)

```bash
# 在你的 AI agent 环境(本机或另一台机器)运行:
python3 -c "
from playwright.sync_api import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.connect_over_cdp('http://localhost:9222')
    print('Connected! Pages:', len(browser.contexts[0].pages))
"
```

输出 `Connected! Pages: 1` 说明 AI agent 跟容器里的 chromium 通了。

## 常用参数

### 自定义分辨率(默认 1280x800x24)

```bash
docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -e DISPLAY_RESOLUTION=1920x1080x24 \
  edik/vnc-research
```

### 自定义 VNC 密码(默认自动生成 8 位数字)

```bash
docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -e VNC_PASSWORD=*** \
  edik/vnc-research
```

### 持久化 Chrome 数据(登录态不丢)

```bash
docker volume create vnc-chrome-data

docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -v vnc-chrome-data:/home/vncuser/.config/chromium \
  edik/vnc-research
```

### 组合使用(推荐)

```bash
docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -e DISPLAY_RESOLUTION=1920x1080x24 \
  -e VNC_PASSWORD=*** \
  -v vnc-chrome-data:/home/vncuser/.config/chromium \
  edik/vnc-research
```

## 容器管理

```bash
# 查看日志
docker logs vnc-research           # 一次性
docker logs -f vnc-research         # 实时跟踪

# 停止 / 启动 / 重启
docker stop vnc-research
docker start vnc-research
docker restart vnc-research

# 删除容器(保留镜像 + volume)
docker stop vnc-research && docker rm vnc-research

# 完全清理(容器 + 镜像 + volume)
docker stop vnc-research && docker rm vnc-research
docker rmi edik/vnc-research
docker volume rm vnc-chrome-data
```

## 更新镜像

```bash
docker pull edik/vnc-research
docker stop vnc-research && docker rm vnc-research
# 再跑 Step 1 的 docker run
```

## 常见问题

### Q:看不到 VNC 画面(黑屏)?
- 等 30 秒(Xvfb + chromium 启动需要时间)
- `docker logs vnc-research` 看有没有 `chromium` 进程报 "Missing X server" 之类
- 重启试试:`docker restart vnc-research`

### Q:VNC 密码忘了?
- `docker logs vnc-research | grep '🔐 VNC 密码'` 找启动时打印的密码
- 重启会自动重新生成:`docker restart vnc-research`

### Q:AI agent 连不上 9222?
- 容器内 `curl http://127.0.0.1:9222/json/version` 验证是否监听(要先 `docker exec -it vnc-research bash`)
- 宿主机 `curl http://localhost:9222/json/version` 看端口映射
- 防火墙是否阻断 9222

### Q:登录小红书/抖音被 datacenter IP 拦?
- 没法解决(本项目不走 stealth 路线)
- 但**已登录账号的 cookie 还在**,agent 用登录态正常访问

### Q:怎么从公网访问?
- 看 [docs/06-外网访问.md](docs/06-外网访问.md)(用 ngrok)
