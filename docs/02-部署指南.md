# 部署指南

> 从一台干净的 Linux 服务器到生产可用,完整步骤。

---

## 前置要求

- Linux 服务器(Debian 12+/Ubuntu 22+/其他主流发行版也行)
- **root 权限**(装包 + 启动 Xvfb)
- **至少 2GB 内存**(chromium + Xvfb)
- **不需要 GPU**
- **不需要物理显示器**

**网络要求**:
- 你的电脑能 SSH 到服务器(管理用)
- 你的电脑能访问服务器 6080 端口(VNC 用)

---

## 方案 A:快速测试(5 分钟)

适合先跑通流程,确认能用,生产部署前过渡用。

### Step 1:装包

```bash
sudo apt update
sudo apt install -y x11vnc novnc websockify x11-apps
```

### Step 2:启动服务

直接用 `scripts/start-all.sh`(本项目提供):

```bash
cd /root/project/docs/vnc-research
chmod +x scripts/*.sh
./scripts/start-all.sh
```

### Step 3:浏览器访问

```
http://你的服务器IP:6080/vnc.html
```

完成。

## 方案 B:生产部署(systemd 服务化)

适合长期使用,**服务器重启自动起**。

### Step 1:装包

```bash
sudo apt update
sudo apt install -y x11vnc novnc websockify x11-apps
```

### Step 2:创建用户(可选,但推荐)

```bash
# 不推荐用 root 跑 chromium,虽然 --no-sandbox 能跑
# 推荐创建专用用户
sudo useradd -m -s /bin/bash vncuser
sudo passwd vncuser
```

### Step 3:systemd 服务文件

#### Xvfb 服务

`/etc/systemd/system/xvfb.service`:

```ini
[Unit]
Description=X Virtual Framebuffer (Xvfb) on :99
After=network.target

[Service]
Type=simple
User=vncuser
Environment=DISPLAY=:99
ExecStart=/usr/bin/Xvfb :99 -screen 0 1280x800x24 -ac +extension GLX +render -noreset
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

#### x11vnc 服务

`/etc/systemd/system/x11vnc.service`:

```ini
[Unit]
Description=x11vnc (VNC Server for Xvfb :99)
After=xvfb.service

[Service]
Type=simple
User=vncuser
Environment=DISPLAY=:99
ExecStart=/usr/bin/x11vnc -display :99 -forever -shared -rfbport 5900 -nopw
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

#### noVNC 服务

`/etc/systemd/system/novnc.service`:

```ini
[Unit]
Description=noVNC + Websockify
After=x11vnc.service

[Service]
Type=simple
User=vncuser
ExecStart=/usr/bin/websockify --web=/usr/share/novnc 6080 localhost:5900
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

#### chromium 服务

`/etc/systemd/system/vnc-chromium.service`:

```ini
[Unit]
Description=Chromium with CDP (for VNC Research)
After=novnc.service

[Service]
Type=simple
User=vncuser
Environment=DISPLAY=:99
Environment=XAUTHORITY=/home/vncuser/.Xauthority
WorkingDirectory=/home/vncuser
ExecStart=/usr/lib/chromium/chromium \
  --no-sandbox \
  --disable-dev-shm-usage \
  --disable-gpu \
  --no-first-run \
  --remote-debugging-port=9222 \
  --remote-debugging-address=127.0.0.1 \
  --user-data-dir=/home/vncuser/.config/chromium \
  --window-size=1280,800 \
  --window-position=0,0 \
  about:blank
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### Step 4:启动 + 开机自启

```bash
sudo systemctl daemon-reload
sudo systemctl enable xvfb x11vnc novnc vnc-chromium
sudo systemctl start xvfb x11vnc novnc vnc-chromium

# 检查状态
sudo systemctl status xvfb x11vnc novnc vnc-chromium
```

### Step 5:健康检查

```bash
# 端口监听
ss -tln | grep -E '5900|6080|9222'

# DevTools 可达
curl http://127.0.0.1:9222/json/version

# 浏览器访问
echo "打开 http://你的服务器IP:6080/vnc.html"
```

## 方案 C:Docker 部署(隔离干净)

适合不想污染主机环境的用户。

### Step 1:写 Dockerfile

`Dockerfile`:

```dockerfile
FROM debian:13-slim

RUN apt-get update && apt-get install -y \
    x11vnc novnc websockify x11-apps chromium \
    python3 python3-pip \
    && rm -rf /var/lib/apt/lists/*

RUN pip3 install --break-system-packages playwright

# VNC 用户
RUN useradd -m -s /bin/bash vncuser

# 复制脚本
COPY scripts/ /usr/local/bin/
RUN chmod +x /usr/local/bin/*.sh

# Xvfb 启动脚本
COPY docker-entrypoint.sh /usr/local/bin/
RUN chmod +x /usr/local/bin/docker-entrypoint.sh

USER vncuser
WORKDIR /home/vncuser

EXPOSE 6080 9222

ENTRYPOINT ["/usr/local/bin/docker-entrypoint.sh"]
```

`docker-entrypoint.sh`:

```bash
#!/bin/bash
set -e

# 启动 Xvfb
Xvfb :99 -screen 0 1280x800x24 -ac +extension GLX +render -noreset &
export DISPLAY=:99

# 等 Xvfb 起来
sleep 2

# 启动 x11vnc
x11vnc -display :99 -forever -shared -rfbport 5900 -nopw &

# 启动 noVNC
websockify --web=/usr/share/novnc 6080 localhost:5900 &

# 启动 chromium
chromium \
  --no-sandbox --disable-dev-shm-usage --disable-gpu \
  --no-first-run \
  --remote-debugging-port=9222 \
  --remote-debugging-address=0.0.0.0 \
  --user-data-dir=/home/vncuser/.config/chromium \
  --window-size=1280,800 \
  about:blank &

# 保持容器跑
tail -f /dev/null
```

### Step 2:构建 + 运行

```bash
docker build -t vnc-research .
docker run -d \
  --name vnc-research \
  -p 6080:6080 \
  -p 9222:9222 \
  -v /tmp/vnc-chrome-data:/home/vncuser/.config/chromium \
  --restart unless-stopped \
  vnc-research
```

### Step 3:浏览器访问

```
http://你的服务器IP:6080/vnc.html
```

## 防火墙配置

**仅内网用**,不需要公网端口。

### iptables(基础)

```bash
# 允许你的内网 IP 段访问 6080
sudo iptables -A INPUT -p tcp --dport 6080 -s 192.168.1.0/24 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 6080 -j DROP

# 9222 只允许本机
sudo iptables -A INPUT -p tcp --dport 9222 -s 127.0.0.1 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 9222 -j DROP
```

### ufw(简易)

```bash
sudo ufw allow from 192.168.1.0/24 to any port 6080
sudo ufw deny 9222
```

### ⚠️ 不要在公网暴露 6080(noVNC 默认无密码!)

如果一定要公网,**必须**:
1. 加 nginx 反向代理 + Basic Auth
2. 或用 cloudflared tunnel + Cloudflare Access
3. 或用 WireGuard VPN 先连内网再访问

详见 [05-安全考量.md](05-安全考量.md)

## 升级与维护

### 更新 chromium

```bash
sudo apt update && sudo apt upgrade chromium
sudo systemctl restart vnc-chromium
```

### 清理 chromium profile(解决登录态问题)

如果 chromium 跑飞了,清 profile 重来:

```bash
sudo systemctl stop vnc-chromium
rm -rf /home/vncuser/.config/chromium
sudo systemctl start vnc-chromium
# 在 VNC 里重新登录各平台
```

### 备份 profile

```bash
# 备份登录态
tar -czf chromium-profile-$(date +%Y%m%d).tar.gz /home/vncuser/.config/chromium
```
