## 坑 1:chromium 启动报 "Missing X server or $DISPLAY"

**症状**:
```bash
[ERROR:ui/ozone/platform/x11/ozone_platform_x11.cc:257] Missing X server or $DISPLAY
[ERROR:ui/aura/env.cc:246] The platform failed to initialize.  Exiting.
```

**根因**: chromium 没拿到 `DISPLAY` 环境变量,默认去找 `:0`(物理显示),但物理显示可能不存在。

**解法**: 启动时显式加 `DISPLAY=:99`:

```bash
DISPLAY=:99 /usr/lib/chromium/chromium ... about:blank
```bash

## 坑 2:chromium 报 "Authorization required, but no authorization protocol specified"

**症状**:
```
Authorization required, but no authorization protocol specified
Error: Can't open display: ":99"
```bash

**根因**: Xvfb 启用了 Xauthority 验证(默认行为),但当前用户没有该 Xauthority。

**解法 A**: Xvfb 启动时加 `-ac`(允许匿名连接):
```bash
Xvfb :99 -screen 0 1280x800x24 -ac +extension GLX +render -noreset
```

**解法 B**: 显式指定 Xauthority:
```bash
XAUTHORITY=/path/to/Xauthority DISPLAY=:99 chromium ...
```bash

**推荐**: 用 Xvfb 启动时加 `-ac`,最简单。

## 坑 3:chromium 进程在但 VNC 看不到画面

**症状**:
- `pgrep -af chromium` 看到进程
- `curl http://127.0.0.1:9222/json/version` 正常返回
- **但 VNC 画面是黑屏**

**根因**: chromium 启动时没拿到正确的 `DISPLAY`,画到了 `:0` 或别的地方,而不是 VNC 共享的 `:99`。

**诊断**:
```bash
cat /proc/<chromium_pid>/environ | tr '\0' '\n' | grep DISPLAY
```

如果显示 `DISPLAY=:0` 或空 → chromium 没画到 :99。

**解法**: 启动时显式加 `DISPLAY=:99`。

## 坑 4:小红书详情页返回 "Sorry, This Page Isn't Available Right Now."

**症状**:
```python
page.goto("https://www.xiaohongshu.com/explore/<hash>") → 404
页面文本: "Sorry, This Page Isn't Available Right Now."
```

**根因**: 小红书笔记的详情页 URL 不是 `/explore/`,而是 `/search_result/<hash>?xsec_token=...`。`xsec_token` 是访问令牌,没有它会被拒。

**解法**: 从搜索结果里抓链接(都带 xsec_token),用这些链接访问详情:

```python
# 错误写法
page.goto(f"https://www.xiaohongshu.com/explore/{note_id}")  # 404

# 正确写法
# 从 page.evaluate 抓到的链接直接用,长这样:
# https://www.xiaohongshu.com/search_result/<hash>?xsec_token=ABC...
page.goto(note_url)  # OK
```bash

## 坑 5:Chromium 启动后 VNC 看不到,但 chromium 进程健康

**症状**: 跟坑 3 类似,但 `DISPLAY` 设置正确。

**根因**: chromium 启动时 `--window-position=0,0` + `--window-size=1280,800` 在 Xvfb 画布 `1280x1024` 上,窗口画在了画布外(800 < 1024,但位置可能错)。

**解法**: `--window-size` 跟 Xvfb 画布**完全一致**:

```bash
# 如果 Xvfb 是 1280x1024
chromium --window-size=1280,1024

# 或者改 Xvfb 大小
Xvfb :99 -screen 0 1280x800x24 ...
chromium --window-size=1280,800
```

## 坑 6:Xauthority 路径在某些命令里被工具过滤

**症状**:
```bash
XAUTHORITY=/tmp/xvfb-run.SWfrmm/Xauthority DISPLAY=:99 xdpyinfo
# 报 Authorization required
```bash

**根因**: 某些 terminal 工具(尤其是 AI agent 用的)有路径过滤,会把 `/tmp/...` 路径替换成 `***`。

**解法**: 用软链把 Xauthority 放到不会被过滤的路径:

```bash
sudo mkdir -p /etc/x11vnc
sudo ln -sf /tmp/xvfb-run.SWfrmm/Xauthority /etc/x11vnc/auth

# 然后用
XAUTHORITY=/etc/x11vnc/auth DISPLAY=:99 xdpyinfo
```

## 坑 7:pip3 install playwright 被 PEP 668 阻挡

**症状**:
```python
error: externally-managed-environment
× This environment is externally managed
```

**根因**: Debian 13 默认禁止 pip 装系统级 Python 包。

**解法**: 加 `--break-system-packages`(本项目是单机部署,可接受):
```bash
pip3 install --break-system-packages playwright
```python

或者用 venv(生产更干净):
```bash
python3 -m venv /opt/vnc-research-venv
source /opt/vnc-research-venv/bin/activate
pip3 install playwright
```

## 坑 8:chromium 启动失败但 stderr 是 "Authorization required"

**症状**:
```bash
Error: Can't open display: (null)
```

**根因**:
- chromium 进程没拿到 `DISPLAY` (坑 1)
- 或者 Xvfb 没在跑
- 或者 Xauthority 错(坑 2)

**诊断**:
```bash
# 1. Xvfb 在跑吗?
pgrep -af Xvfb

# 2. chromium 进程的 DISPLAY?
cat /proc/<pid>/environ 2>/dev/null | tr '\0' '\n' | grep DISPLAY

# 3. Xvfb 接受连接吗?
DISPLAY=:99 XAUTHORITY=*** xdpyinfo
```bash

## 坑 9:websockify 启动后 6080 端口能开,但 noVNC 报连接错

**症状**:
- `http://服务器:6080/vnc.html` 能打开
- 但页面里 noVNC 报 "Server disconnected" 或 "Connection failed"

**根因**: x11vnc 没在 5900 跑 / 防火墙拦了 / websockify 连不上 5900

**诊断**:
```bash
# 1. x11vnc 在跑吗?
pgrep -af x11vnc

# 2. 5900 监听吗?
ss -tln | grep 5900

# 3. websockify 日志?
journalctl -u novnc -n 50
```

## 坑 11:Chromium 反复崩(OOM / GPU 错误)

**症状**: chromium 跑几小时就崩,systemd 自动重启。

**根因**:
- `--disable-gpu` 没加
- 服务器内存不足(<2GB)
- chromium 有 bug

**解法**:
```bash
# 必加的稳定参数
chromium \
  --no-sandbox \
  --disable-dev-shm-usage \  # 避免 /dev/shm 问题
  --disable-gpu \             # Xvfb 没 GPU
  --no-first-run \            # 跳过首次设置向导
  --disable-extensions \      # 不要扩展
  ...
```bash

## 坑 12:VNC 看到 chromium 但操作卡顿

**症状**: VNC 画面刷新慢,鼠标点击延迟。

**根因**:
- 网络慢(尤其是远距离)
- chromium 太占 CPU
- `x11vnc -ncache 10` 没启用(默认就开了)

**解法**:
```bash
# 加压缩
x11vnc -display :99 -forever -shared -rfbport 5900 -nopw \
       -compress 9 -quality 5

# 或者换 tight encoding
x11vnc -display :99 -forever -shared -rfbport 5900 -nopw \
       -tight
```

## 坑 13:Playwright `evaluate` 里字符串含 `\\` 被过滤

**症状**:
```python
page.evaluate("() => Array.from(document.querySelectorAll('a[href*=\"search_result/\"]'))")
# 报 JS 语法错(因为 \" 被过滤了)
```python

**根因**: terminal 工具把 `\\` 当转义处理。

**解法**: 用 raw string + 单引号外层:
```python
script = r'''
() => {
    const links = document.querySelectorAll('a[href*="search_result/"]');
    return Array.from(links).map(a => a.href);
}
'''
page.evaluate(script)
```

或者用 `subprocess.run(['python3', '-c', script])` 跑多行脚本最稳。

## 坑 14:VNC 浏览器登录了小红书,但 agent 抓不到数据

**症状**: 你在 VNC 登录了小红书,agent 用 playwright 打开搜索页,内容还是"登录后查看"。

**根因**: chromium profile 没生效 / 你登录的 chromium 实例 ≠ agent 连的实例。

**诊断**:
```bash
# 看 chromium 进程实际用的 user-data-dir
pgrep -af chromium | grep -o "user-data-dir=[^ ]*"
```bash

**解法**: agent 连的 9222 必须对应**你登录过的那个 chromium 实例**。如果起了多个 chromium,会有多个 9222 端口,确保连的是同一个。

## 坑 15:8080 端口被占用(误用 8080 而不是 6080)

**症状**: noVNC 启动失败,提示端口占用。

**根因**: 用了 8080(常见 web 端口)而不是 6080(noVNC 默认)。

**解法**: 用 6080(或任意空闲端口):
```bash
websockify --web=/usr/share/novnc 6080 localhost:5900
```

## 如果还是有问题

1. **看 `scripts/health-check.sh` 输出** — 99% 的问题能定位
2. **看 systemd 日志** — `journalctl -u <service> -n 100`
3. **重新跑 `scripts/start-all.sh`** — 重置整个状态
4. **最后手段:重启服务器** — 清理所有状态
