# 部署指南

> 标准部署方式:用 Docker Hub 官方镜像 [`edik/vnc-research`](https://hub.docker.com/r/edik/vnc-research),一行命令起服务。

镜像在独立仓库 [vnc-research-docker](git@git.edik.cn:edik-project/vnc-research-docker.git) 维护(本仓库**不再保留** Dockerfile / scripts)。

---

## 前置要求

- **Linux 服务器**(Debian 12+/Ubuntu 22+ 等主流发行版都行)
- **Docker 引擎** ≥ 20.10(`docker --version` 自查)
- **至少 2GB 可用内存**(chromium + Xvfb)
- 不需要 GPU / 不需要物理显示器

**网络要求**:
- 你的电脑能 SSH 到服务器(管理用)
- 你的电脑能访问服务器 6080 端口(VNC 用)

---

## 唯一方式:Docker 镜像(推荐)

### 快速启动

```bash
docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  --restart unless-stopped \
  edik/vnc-research

# 浏览器打开 http://<host>:6080/vnc.html
```

看到 noVNC 桌面 = 成功。

> 默认把 6080(noVNC Web UI)+ 9222(Chromium DevTools)都映射到宿主机。
> 如需从公网访问 → 看 [docs/03-外网访问.md](docs/03-外网访问.md),用 ngrok 隧道。

### 自定义分辨率

```bash
docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -e DISPLAY_RESOLUTION=1920x1080x24 \
  edik/vnc-research
```

支持的格式:`<宽>x<高>x<色深>`,默认 `1280x800x24`。

### 完整环境变量

| env | 默认值 | 含义 |
|---|---|---|
| `DISPLAY_RESOLUTION` | `1280x800x24` | 虚拟桌面分辨率 + 色深 |
| `CHROMIUM_FLAGS` | 空 | 给 chromium 加额外启动参数,空格分隔 |

### 更新镜像

```bash
docker pull edik/vnc-research
docker stop vnc-research && docker rm vnc-research
# 再跑上面的 docker run 命令
```

> 数据在 volume 里,不会丢;只换容器。

### 查看日志 / 调试

```bash
# 实时日志
docker logs -f vnc-research

# 进容器
docker exec -it vnc-research bash

# 健康检查:确认 6080 / 9222 端口
docker port vnc-research
```

## 数据持久化

chromium 的 profile / cookies / 登录态 / 扩展 默认存在容器内 `/root/.config/chromium`,**容器删了就丢**。下面 3 种持久化方式选一个。

### 方式 A:named volume(Docker 自动管理,推荐)

```bash
docker volume create vnc-chrome-data

docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -v vnc-chrome-data:/root/.config/chromium \
  edik/vnc-research
```

- 优点:Docker 自动管理,不用关心 host 路径
- 缺点:数据在 `/var/lib/docker/volumes/vnc-chrome-data/_data/`,想直接看文件不太方便
- 重启容器:**不会丢**

### 方式 B:bind mount(直接挂 host 目录)

```bash
mkdir -p ~/vnc-chrome-data

docker run -d --name vnc-research \
  -p 6080:6080 -p 9222:9222 \
  -v ~/vnc-chrome-data:/root/.config/chromium \
  edik/vnc-research
```

- 优点:数据直接在 host,可读可备份
- 缺点:删 host 目录 = 丢数据
- 重启容器:**不会丢**(镜像启动时会自动清 SingletonLock,见下)

### 方式 C:docker compose(最简单)

`docker-compose.yml`:

```yaml
services:
  vnc-research:
    image: edik/vnc-research:latest
    container_name: vnc-research
    restart: unless-stopped
    ports:
      - "6080:6080"
      - "9222:9222"
    volumes:
      - vnc-chrome:/root/.config/chromium

volumes:
  vnc-chrome:
```

跑:

```bash
docker compose up -d
```

完整 compose file 参考 [vnc-research-docker 仓库](git@git.edik.cn:edik-project/vnc-research-docker.git) 的 `docker-compose.yml`。

### 注意事项:bind mount + 容器重启

如果用 **方式 B**(bind mount),第一次启动后删容器再启动会看到:

```
ERROR: The profile appears to be in use by another Chromium process (XXX) on another computer (ebd0e1c86af6). Chromium has locked the profile so that it doesn't get corrupted.
```

**原因**:chromium 在 profile 目录留了 `SingletonLock / SingletonCookie / SingletonSocket` 三个文件,记录旧容器的 hostname。容器删了但 host 目录保留,新容器启动时新 chromium 看到旧 hostname 锁就拒绝启动。

**解决**:**v0.1.13+ 镜像已自动处理**(启动前自动 rm 这三个文件)。如还在用更老镜像,手动清:

```bash
rm ~/vnc-chrome-data/Singleton{Lock,Cookie,Socket}
```

named volume 不受影响(Docker 自动管,不跨容器复用 hostname)。

## 防火墙 / 网络

- 6080(noVNC Web UI)是**唯一**需要对外暴露的端口
- 9222(Chromium DevTools)只给**本机 / 内网**用,**不要**暴露到公网
- 如需从公网访问 6080 → 见 [docs/03-外网访问.md](docs/03-外网访问.md),用 ngrok 隧道

### 内网限制 6080 访问源(可选)

```bash
# ufw:只允许 192.168.1.0/24 访问
sudo ufw allow from 192.168.1.0/24 to any port 6080
sudo ufw deny 9222
```

```bash
# iptables:同上
sudo iptables -A INPUT -p tcp --dport 6080 -s 192.168.1.0/24 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 6080 -j DROP
sudo iptables -A INPUT -p tcp --dport 9222 -s 127.0.0.1 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 9222 -j DROP
```

## 常见问题

### 容器起不来 / 端口冲突

```bash
# 看日志
docker logs vnc-research

# 6080 / 9222 被占?
ss -tln | grep -E '6080|9222'
```

把宿主机占用 6080/9222 的进程停掉,或换端口映射(`-p 16080:6080`)。

### 进 VNC 看到黑屏

- 等 5-10 秒(chromium 还在启动)
- 看 `docker logs -f vnc-research` 确认 chromium 已起来
- 刷新浏览器 / 清 noVNC 缓存

### 9222 连不上 / playwright 报错

- 确认容器里 9222 起来了:`docker exec vnc-research ss -tln | grep 9222`
- 确认宿主机 9222 通了:`curl http://127.0.0.1:9222/json/version`
- 9222 监听在 `0.0.0.0`(容器内),但只映射到 `127.0.0.1` 的话外网不通,改成 `-p 9222:9222` 映射到 `0.0.0.0`

### 想清掉登录态重来

```bash
docker stop vnc-research && docker rm vnc-research
docker volume rm vnc-chrome-data
# 重新跑 docker run
```

### 想升级 chromium 版本

直接 `docker pull edik/vnc-research:latest` 拿新镜像,旧镜像 tag 会留作回滚。
