# 安全考量

> VNC + noVNC 默认无密码,**暴露在公网 = 任何人能直接进你的服务器桌面**。
> 这份文档讲清楚:哪些场景安全,哪些必须加固。

---

## 风险矩阵

| 暴露方式 | 风险等级 | 说明 |
|---------|---------|------|
| **仅内网**(192.168.x.x / 10.x.x.x) | 🟢 低 | 默认配置即可,假设你的内网是可信的 |
| 内网 + 公司员工可访问 | 🟡 中 | 同事能看到你的 VNC(可能看到登录态) |
| 暴露公网 + 无密码 | 🔴 **极高** | 任何人扫到 6080 端口都能进 |
| 暴露公网 + 加 nginx Basic Auth | 🟡 中 | 防大部分扫描,但 Basic Auth 不抗暴力 |
| 暴露公网 + 加 Cloudflare Access | 🟢 较好 | 强身份验证,推荐 |
| 暴露公网 + 加 WireGuard VPN | 🟢 好 | 跟"内网"一样安全,推荐 |

## 暴露前必读

### ❌ 绝对不要这样做

```bash
# 错:在云服务器上开公网 6080,无密码
x11vnc -display :99 -nopw ...  # -nopw = 无密码!
# 然后开放 0.0.0.0:6080 到公网
# = 任何人能直接进你的桌面,看到你的所有登录态
```bash

### ✅ 推荐方案(按优先级)

#### 方案 A:仅内网(最简单,推荐)

如果你家/公司网络是可信的:
1. 服务器只在**内网 IP** 上监听(noVNC 默认就是 0.0.0.0,但只在内网用)
2. 用防火墙限制 6080 只能从你的 IP 段访问
3. x11vnc `-nopw` 可以接受(只在可信内网)

**配置示例**(防火墙):
```bash
# 仅允许 192.168.1.0/24 访问 6080
sudo iptables -A INPUT -p tcp --dport 6080 -s 192.168.1.0/24 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 6080 -j DROP
```

#### 方案 C:Cloudflare Tunnel + Access(推荐生产)

```bash
# 1. 装 cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb

# 2. 登录
cloudflared tunnel login

# 3. 创建隧道
cloudflared tunnel create vnc-research

# 4. 配置 DNS
cloudflared tunnel route dns vnc-research vnc.example.com

# 5. 启动(转发到本地 6080)
cloudflared tunnel --url http://localhost:6080 vnc-research

# 6. (可选) 在 Cloudflare Zero Trust 加 Access 策略
# 要求 Google/GitHub 登录才能访问
```

#### 方案 D:WireGuard VPN(最安全,稍微麻烦)

```bash
# 1. 服务器装 wireguard
sudo apt install -y wireguard

# 2. 生成密钥
wg genkey | tee /etc/wireguard/server.key | wg pubkey > /etc/wireguard/server.pub

# 3. 配置 /etc/wireguard/wg0.conf
[Interface]
Address = 10.0.0.1/24
ListenPort = 51820
PrivateKey = <server.key 内容>

[Peer]
PublicKey = <你的电脑公钥>
AllowedIPs = 10.0.0.2/32

# 4. 启动
sudo systemctl enable --now wg-quick@wg0

# 5. 你的电脑装 wireguard 客户端,导入配置,连接
# 6. 连上后,你的电脑就有 10.0.0.2 地址,访问服务器就像在内网
```text

## 隐私风险

即使部署在安全的内网,这套架构仍有一些隐私风险需要知道:

### 1. cookie 和登录态

- chromium profile 里包含**所有平台的 cookie 和 localStorage**
- 任何能访问 VNC 的人都能看到你的登录态
- **不要在 VNC 里登录敏感账号**(银行、邮箱、生产数据库)

### 2. AI agent 看到的页面

- agent 用 playwright 能读 chromium 里**所有打开的页面**
- 包括你之前在 VNC 手动打开的标签页
- **不要在 VNC 里打开私人内容**(微信 Web、银行、邮箱)

### 3. AI agent 提交的数据

- agent 抓的数据落本地 `/tmp/`
- 不会上传到任何第三方
- 但**agent 自己在网络上的操作是公开的**(用你的 IP 访问平台)

### 4. VNC 录屏

- noVNC 是浏览器渲染,不在服务器端录像
- **没有持续录屏**(不像 Zoom / Teams)
- 但 x11vnc 本身可以加 `-record` 录像,**别用**

## 多人协作场景

### 场景 A:你一个人用

✅ 最简单。安全配置跟上面一样。

### 场景 B:你 + 朋友共享服务器

⚠️ 需要:
- 每个人单独的 chromium profile(`--user-data-dir=/home/user1/.config/chromium`)
- 每个人单独的 VNC 端口(`x11vnc -rfbport 5901`)
- 每个人单独的 DevTools 端口(`--remote-debugging-port=9223`)
- **每个人独立登录的账号**才安全(否则互相能看到对方登录态)

### 场景 C:团队 5+ 人

**不建议用本方案**。考虑:
- Kasm Workspaces(商业 SaaS,按月付费)
- Apache Guacamole(Web VNC 多用户)
- Selenium Grid(自动化专用)

## 数据保留

| 数据 | 保留位置 | 建议 |
|------|---------|------|
| chromium profile | `$HOME/.config/chromium/` | **必须本地,绝不上传** |
| VNC 临时文件 | `/tmp/.X11-unix/` | 重启自动清,无需手动 |
| agent 抓取数据 | `/tmp/*.json` | 7 天后清 |
| 正式报告 | `/root/project/docs/` | 长期保留,但**脱敏**(不写账号密码) |

## 审计和日志

### 想看谁什么时候用了 VNC

x11vnc 默认有日志,加 `-verbose`:
```bash
x11vnc -display :99 -forever -shared -rfbport 5900 -nopw -verbose
```

日志写到 `~/.vnc/*log`。

### 想看 agent 什么时候做了什么操作

agent 用 `subprocess.run(['python3', '-c', script])` 跑脚本,**没自动日志**。
建议加显式 print,或写到 `/var/log/vnc-research-agent.log`。

---

## 一句话总结

> **默认只内网用,绝对不要公网暴露 6080(无密码)。**
> 如果必须公网,加 Cloudflare Tunnel + Access。
